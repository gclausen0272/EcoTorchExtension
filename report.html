Reduction methodologies

  • Quantization: At training time, we must do the resource intensive task of computing gradients, however when it comes to testing, we can get away with a less memory and storage intensive representation for our model parameters. Quantizing to INT8 or FP16 allows a model to perform much faster, expending lower energy with a minimal loss in accuracy.
  • Pruning/Sparsity: If you have a hammer, every problem seems like a nail. In deep learning, we tend to use more parameters just in case they perform better, than use a lesser number of parameters and under-perform at the objective. Once we have an optimized model, we could then prune parameters that are not very important based on a heuristic to make the model faster and memory efficient.

    The good news is there's a lot that you can do to help!

  • Purchase carbon offsets: carbon offsets are a great way to balance the emissions you create through machine learning. Helpful tools such as wren.co and goldstandard are great resources
  • Use clean energy: consider switching your home or organization to more sustainable and clean methods of power
  • Run local: running code locally tends to be more sustainable than running with cloud services.
  • Be more informed: using tools such as ours can raise awareness of the ecological impacts of ML and motivate you to make more sustainable decisions
